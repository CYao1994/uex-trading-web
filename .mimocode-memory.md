## Architecture decisions
_Major design choices with rationale. The "why" matters more than the "what" for future sessions._

- **Three-layer caching**: frontend localStorage (30min) → EdgeOne KV (1-24h) → backend in-memory TTLCache (2-24h). Rationale: reduce UEX Corp API calls, improve latency for Chinese users.
- **Edge Functions as cache proxy**: EdgeOne Edge Functions check KV cache before forwarding to Python Cloud Function. POST requests (route planning) bypass cache.
- **Python backend uses urllib.request** (not httpx/curl) because EdgeOne Cloud Functions may lack curl binary.
- **All Chinese name mappings** maintained in `data_mapper.py` (960+ lines), auto-synced via `scripts/sync_chinese_names.py` from ParaTranz and Gitee.
- **EdgeOne Cloud Functions不支持相对import**: `from .xxx import ...`在EdgeOne Python打包环境中无法解析，必须使用绝对import `from api.xxx import ...`或`from services.xxx import ...`。index.py中`sys.path.insert(0, backend_dir)`已设置。`__pycache__/`残留旧.pyc也会干扰新文件的import。
- **战争债券数据源RSI GraphQL API**: 替代starnotifier.com HTML爬取，覆盖所有分类(Gear/Packs/Add-Ons/Standalone Ships/Package)。数据结构从`{ccu_items, standalone_ships}`改为`{categories: {"Gear": [...], ...}}`。前端WarbondPanel动态遍历categories渲染。
- **战争债券数据源升级为RSI升级GraphQL API(2026-06-18)**: 主数据源从`/graphql`(RSI Store GraphQL)切换为`/pledge-store/api/upgrade/graphql`(RSI升级GraphQL)，免认证公开端点，返回所有船+warbond SKU+图片(`storeThumbSkuDetail`)。starnotifier降级为仅升级API失败时的fallback。RSI旧GraphQL仅用于Gear/Packs/Package/Add-Ons等非船舰分类。**⚠️ 局限性**: `limitedTimeOffer=True`返回24个warbond但实际只有8个在售，后端worker筛选逻辑未开源。
- **Warbond schema与scraper必须同步**: schemas.py的WarbondResponse/WarbondItem必须匹配warbond_scraper.py返回格式。修改scraper数据结构后必须同步更新Pydantic模型。
- **ParaTranz translate()非重叠贪心替换**: 收集所有有效匹配→按长度降序→贪心非重叠替换(occupied set防重叠)。比单最长匹配更完整(compound名多子串覆盖)，比链式replace更安全(无连锁损坏)。3条过滤规则剔除垃圾数据。**注意**: 单最长匹配方案对compound名覆盖不全，链式replace会损坏字符串。
- **所有翻译统一ParaTranz来源**: 用户要求"所有模块中的中文均来自汉化项目文件"。所有翻译函数(get_terminal_zh, get_commodity_zh, get_vehicle_zh, _get_name_zh)统一ParaTranz优先，硬编码字典仅作fallback。硬编码字典(SHIP_NAME_ZH/TERMINAL_ZH_MAP等)保留但仅在ParaTranz无翻译时使用。
- **translate()厂商前缀拼接**: 精确匹配后尝试`_MANUFACTURER_EN`前缀拼接（如"Anvil"+"C8 Pisces"→"Anvil C8 Pisces"→"铁砧 C8 双鱼座"），优先返回更长更完整的翻译。ParaTranz用"铁砧"(字面anvil)而非"安维尔"(音译)。
- **前端SWR缓存模式**: cache.js `getStale()`返回`{data, expired}`（过期也返回），client.js `cachedGet()`根据stale结果决定：缓存有效→返回+后台_bgRefresh()（fire-and-forget，错误静默）；缓存过期→返回stale数据+后台刷新；无缓存→同步请求。不使用useSWR hook，保持简单axios封装。
- **EdgeOne Pages部署分支是main**: 推送代码必须同时推送到`master`和`main`两个分支: `git push origin master master:main`。只推master不触发部署。
- **大组件拆分模式(2026-06-24)**: 6个超大组件(648-1091行)拆分为24+子组件。规则：纯渲染逻辑提取到子组件，所有useState/useEffect/useCallback留在父组件。子组件通过props接收数据和回调。每个新文件需要独立import（React/MUI/icons）。拆分后常量（如PORT_TYPE_ZH）从父组件删除避免unused warning。子组件命名用`组件名+功能`格式（如FilterBar/ItemCard）。
- **HTTPException只在顶部import**: routes.py中不要在except块内局部import HTTPException，直接在文件顶部全局import即可
- **resolve_terminal用dict查找**: uex_api.py中`resolve_terminal(tid)`用`_terminal_id_map` dict延迟初始化+O(1)查找，替代线性扫描遍历全部terminals
